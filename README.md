# LLM Triage Tool

## Overview

The **LLM Triage Tool** is an AI-assisted security analysis microservice developed as part of the **PipelineFortress** DevSecOps Security Pipeline.

The service receives security findings generated by multiple security scanners, consolidates and analyzes them, removes duplicate findings, prioritizes risks, redacts sensitive information, and leverages a Large Language Model (LLM) to generate concise remediation recommendations for developers.

The microservice is implemented using **FastAPI** and is designed to integrate seamlessly into a GitHub Actions CI/CD pipeline.

---

# Problem Statement

Modern DevSecOps pipelines typically execute multiple security scanners such as:

* Semgrep (SAST)
* Trivy (Filesystem & Container)
* OWASP ZAP (DAST)
* Gitleaks (Secrets Detection)
* Checkov (Infrastructure as Code)
* Syft (SBOM Generation)

Although these tools provide comprehensive security coverage, they often generate:

* Duplicate findings
* Large volumes of alerts
* Scanner-specific report formats
* Limited remediation guidance
* High false-positive rates

Security engineers must manually review and prioritize these findings before developers can remediate them.

The LLM Triage Tool automates this process.

---

# Features

* Consolidates findings from multiple security scanners
* Normalizes different scanner output formats
* Removes duplicate findings
* Prioritizes vulnerabilities based on severity
* Redacts secrets and proprietary source code before sending data to an LLM
* Generates AI-assisted vulnerability explanations
* Suggests secure remediation in 3–5 developer-friendly lines
* Maps findings to OWASP Top 10 and CWE (where applicable)
* Produces Markdown and JSON reports
* Supports GitHub Actions integration
* Supports human approval before code remediation

# Workflow

1. Receive a consolidated security report generated by the CI/CD pipeline.
2. Validate the incoming request.
3. Normalize findings into a common format.
4. Redact secrets, tokens, credentials, proprietary identifiers, and large source-code blocks.
5. Cluster duplicate findings reported by multiple scanners.
6. Prioritize findings based on severity and risk.
7. Send only the sanitized findings to the LLM.
8. Generate:

   * Business impact
   * Root cause
   * Suggested remediation
   * Developer summary
9. Validate the LLM response.
10. Generate JSON and Markdown reports.
11. Produce draft remediation suggestions for human review.

# Security Features

## Redaction Layer

Before any prompt is sent to the LLM, the service removes sensitive information including:

* API Keys
* Passwords
* JWT Tokens
* AWS Access Keys
* Private Keys
* Email Addresses
* IP Addresses
* Proprietary Identifiers
* Large Source-Code Blocks

This ensures confidential project information never leaves the build runner.

---

# Human-in-the-Loop

The LLM does **not** automatically modify application source code.

For the highest-priority findings, the service generates remediation suggestions that are submitted as **Draft Pull Requests**. A human reviewer must approve any changes before they are merged.

This follows secure AI governance principles by ensuring AI-generated recommendations are reviewed prior to deployment.
